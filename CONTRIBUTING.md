# Contributing

Thank you for helping improve this course.

## How to contribute
1. Open an issue to describe your idea or problem.
2. If you are changing files, make a new branch.
3. Keep changes small and focused.
4. Add or update docs when needed.

## Style guide
- Use clear and simple language.
- Keep lessons short and focused.
- Use file and folder names already in the plan.

## Reporting problems
Please include:
- What you expected to happen
- What actually happened
- A screenshot or file link if possible

## Code of conduct
By taking part, you agree to follow `CODE_OF_CONDUCT.md`.
